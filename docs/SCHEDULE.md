# Расписание преподавателя (`schedule.txt`)

Отдельный **текстовый файл-расписание**, который преподаватель заполняет вручную.
Приложение читает его и генерирует занятия в БД — единый источник истины по занятиям
остаётся таблица `lessons` (см. `database.py`).

## Как пользоваться

1. Отредактируйте файл `schedule.txt` (лежит рядом с приложением).
2. На экране **«Сегодня»** нажмите **«📅 Загрузить расписание»**.
3. Приложение разберёт файл, сгенерирует занятия и заменит прежние **запланированные**
   (`scheduled`) занятия. **Проведённые (`held`) и отменённые (`cancelled`) не трогаются** —
   учёт сохраняется.

## Формат файла

- Кодировка UTF-8. Разделитель полей — `;`. Строки с `#` и пустые игнорируются.
- **Мета-строка** (обязательна) задаёт точку отсчёта чётности недель:

  ```
  СЕМЕСТР=2026-09-07; ПЕРВАЯ_НЕДЕЛЯ=Н
  ```

  - `СЕМЕСТР` — дата начала семестра (ГГГГ-ММ-ДД).
  - `ПЕРВАЯ_НЕДЕЛЯ` — чётность недели, в которую попадает дата семестра: `Ч` или `Н`.

- **Строки занятий**: `День; Пара; Предмет; Группа; Ч/Н`

  ```
  Понедельник; 1; Математика; ИС-11; ЧН
  Среда;       2; Математика; ИС-11; Н
  ```

  - **День** — `Понедельник…Воскресенье` (можно сокращённо `Пн`, `Вт`, …).
  - **Пара** — только **номер** пары (1..8), без времени. Преподаватель сам знает,
    во сколько какая пара. «**Окна**» — это просто пропущенные номера (например есть
    пара 1 и 3, а 2 нет).
  - **Предмет** и **Группа** — должны точно совпадать с карточкой предмета в приложении
    (иначе строка будет пропущена и показана в отчёте о загрузке).
  - **Ч/Н** — признак недели:
    - `ЧН` — предмет идёт **каждую** неделю в этот день/пару;
    - `Ч`  — только по **чётным** неделям (через неделю);
    - `Н`  — только по **нечётным** неделям (через неделю).

## Сколько занятий генерируется

Для каждого предмета генерируется ровно **план из карточки предмета + 1** занятий,
начиная с даты семестра, по его слотам с учётом Ч/Н. Все они создаются со статусом
`scheduled` и появляются на экране **«Расписание»** в соответствующие даты.

> **Важно про даты.** Занятия создаются начиная с даты `СЕМЕСТР` (а не «на сегодня»).
> Если сегодняшний день — вне семестра (например, каникулы), на текущую дату занятий
> не будет. Поэтому на экране «Расписание» есть **навигация по датам** (стрелки `‹ ›`
> и кнопка «Сегодня»), а сразу после загрузки экран **автоматически перепрыгивает на
> дату первого сгенерированного занятия** и сообщает её в отчёте о загрузке.

## Время пар (только для экспорта в календарь)

В расписании времени **нет** — только номер пары. Но при выгрузке `.ics` в Google/Яндекс
Календарь нужно конкретное время, поэтому принято:

- одна пара длится **1 час 45 минут** (105 мин);
- пары идут подряд от **09:00**:

  | Пара | Начало | Конец |
  |------|--------|-------|
  | 1    | 09:00  | 10:45 |
  | 2    | 10:45  | 12:30 |
  | 3    | 12:30  | 14:15 |
  | 4    | 14:15  | 16:00 |

Реальные «окна» между парами на эти расчётные времена не влияют — они нужны только
для события в календаре.

## Где что в коде

- `schedule_file.py` — разбор файла и генерация занятий (headless, без БД и Kivy).
- `database.py` — `subject_plans_map()`, `find_subject_id()`, `replace_scheduled_lessons()`.
- `main.py` — кнопка «📅 Загрузить расписание» на экране «Сегодня».
- `calendar_export.py` — `LESSON_DURATION_MIN = 105` (длительность пары для `.ics`).
