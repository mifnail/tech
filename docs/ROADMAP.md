# ROADMAP — LessonTracker

Работаем **сверху вниз**. Каждый пункт доводится до рабочего состояния,
покрывается тестами (где применимо) и коммитится, прежде чем брать следующий.

## Вертикальные срезы
- [ ] **v0.1 — Слой БД + тесты.** `database.py`: схема (предметы, группы, студенты,
      занятия, отметки), CRUD, уникальность данных. Зелёные тесты в CI. ← *текущий*
- [x] **v0.2 — Главный экран.** Список предметов; прогресс проведённых/оставшихся
      занятий — прогресс-баром на фоне за названием предмета (`canvas.before`).
- [x] **v0.3 — Ввод предмета/группы/студентов.** В т.ч. массовый ввод студентов
      (каждый с новой строки) и выбор уже существующих (неповторяемость).
- [~] **v0.4 — Проведение занятия.** Напоминание раз в час → «Занятие проведено?» →
      отметка присутствующих нажатием на ФИО (цикл: отсутствует → ✓ → оценка 5→4→3→2),
      неотмеченные = отсутствующие. Код готов (`main.py`: LessonScreen) — ожидает локальной проверки человеком.
- [ ] **v0.4 — Проведение занятия.** Напоминание раз в час → вопрос «занятие проведено?»
      → отметка присутствующих нажатием на ФИО (галочка/оценка); неотмеченные = отсутствуют.
- [~] **v0.4a — Расписание на сегодня + статусы занятий.** Отдельный от сводного
      главного экран «Сегодня»: занятия за сегодня по всем предметам (несколько
      занятий/день по одному предмету, в т.ч. параллельные). Статусы занятия:
      `scheduled` / `held` (только оно в учёте) / `cancelled` (не учитывается).
      Тап → «Проведено?»: Да → отметка; Нет → заменить на другой предмет / отменить.
      Кнопка «➕ ДОБАВИТЬ ЗАНЯТИЕ» (внеочередное) + экспорт занятий из БД в `.ics`.
      Код готов (`main.py`: TodayScheduleScreen, `calendar_export.py`) — ожидает
      локальной проверки человеком.
- [~] **v0.5 — Ведомость успеваемости.** Экран «Ведомость» (выбор предмета):
      диаграмма среднего балла по студентам средствами Kivy `canvas` (без matplotlib)
      + пофамильно (средний арифметический балл, число оценок, посещаемость) +
      сводка «план / проведено / осталось» по занятиям. Учитываются только занятия
      со статусом `held`. Код готов (`main.py`: GradebookScreen, GradeBarChart;
      `database.py`: subject_summary, subject_gradebook) — ожидает проверки человеком.
- [~] **v0.6 — Календарь + выгрузка ведомости.** (1) Экспорт расписания из БД в
      `.ics` (импорт в Google и Яндекс Календарь) — `calendar_export.py`, кнопка
      «📤 В календарь (.ics)» на экране «Сегодня». Прямая синхронизация с Google
      Calendar API УБРАНА (по решению владельца — оставляем только `.ics`).
      (2) Выгрузка ведомости в Excel (`openpyxl`) и PDF (`reportlab`) —
      `report_export.py`, кнопки на экране «Ведомость» (`docs/REPORTS.md`). Тяжёлые
      зависимости грузятся лениво и не входят в APK (кроме pure-python openpyxl).
      Код готов — ожидает проверки человеком.

- [~] **v0.7 — Расписание из файла.** Отдельный текстовый файл `schedule.txt`,
      заполняемый преподавателем вручную; кнопка «📅 Загрузить расписание» на экране
      «Сегодня» разбирает его и генерирует занятия в БД (по предмету — план+1), с
      учётом номеров пар (без времени, «окна» = пропуски) и признака чёт/нечёт (Ч/Н).
      Время пары для `.ics` — 1:45 от 09:00 (`schedule_file.py`, `docs/SCHEDULE.md`).
      Headless-тесты зелёные; UI-загрузка ожидает проверки человеком.

## Правила
- Не начинаем следующий пункт, пока текущий не «зелёный» и не закоммичен.
- **Каждый этап запускается локально и проверяется человеком** перед commit/push.
- Работающий код не рефакторим без падающего теста (см. AGENT.md §5).
- Тестовая БД создаётся через `python seed_db.py` (файл `lessons.db` коммитится в git).
