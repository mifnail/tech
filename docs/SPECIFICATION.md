# Спецификация системы «Учёт занятий» (TeachHelper4)

## 1. Назначение системы
Мобильное приложение (Android APK) для преподавателя колледжа/вуза для:
- Ведения журнала проведённых занятий
- Учёта посещаемости и успеваемости студентов
- Контроля выполнения учебного плана (план/факт часов)
- Учёта замен предметов

## 2. Технический стек (заморожен)
| Компонент | Технология |
|-----------|-----------|
| Язык | Python 3 |
| Бэкенд | Flask (REST API) |
| Фронтенд | Vanilla JS SPA (без фреймворков) |
| База данных | SQLite (через `sqlite3`) |
| Мобильная сборка | WebView (python-for-android) / Buildozer |
| CI/CD | GitHub Actions (тесты + APK) |

## 3. База данных (ER-диаграмма)

### 3.1 Схема (английские названия)

```
┌──────────────┐       ┌─────────────────┐
│   groups     │       │    subjects     │
├──────────────┤       ├─────────────────┤
│ id (PK)      │──┐    │ id (PK)         │──┐
│ name (UNIQUE)│  │    │ name            │  │
└──────────────┘  │    │ total_hours     │  │
                  │    │ group_id (FK)   │←─┘
                  │    └────────┬────────┘
                  │             │
                  │    ┌────────┴────────┐
                  │    │   schedule      │
                  │    ├─────────────────┤
                  │    │ id (PK)         │
                  │    │ day_of_week     │
                  │    │ lesson_number   │
                  │    │ subject_id (FK) │── FK → subjects.id
                   │    │ week_type       │  (0=кажд,1=нечет,2=чет)
                  │    └─────────────────┘
                  │
┌──────────────┐  │    ┌─────────────────┐
│   students   │  │    │    lessons      │
├──────────────┤  │    ├─────────────────┤
│ id (PK)      │  │    │ id (PK)         │
│ group_id (FK)│←─┘    │ subject_id (FK) │── FK → subjects.id (плановый)
│ last_name    │       │ actual_subj (FK)│── FK → subjects.id (фактический)
│ first_name   │       │ date            │
│ middle_name  │       │ status          │  ('held' | 'free')
└──────────────┘       └────────┬────────┘
     │                          │
     │    ┌─────────────────┐   │
     │    │    grades       │   │
     │    ├─────────────────┤   │
     └───→│ id (PK)         │   │
          │ lesson_id (FK)  │←──┘
          │ student_id (FK) │── FK → students.id
          │ grade           │  ('0','2','3','4','5','absent')
          └─────────────────┘
```

### 3.2 Таблицы

| Таблица | Назначение | Ключевые поля |
|---------|-----------|---------------|
| `groups` | Учебные группы | name (уникальное) |
| `students` | Студенты | group_id → groups, ФИО |
| `subjects` | Предметы | name, total_hours (план часов), group_id → groups |
| `schedule` | Шаблон расписания | day_of_week (1-6), lesson_number, subject_id, week_type |
| `lessons` | Факты проведения занятий | subject_id (план), actual_subject_id (факт), date, status |
| `grades` | Оценки | lesson_id, student_id, grade |

### 3.3 Статусы занятий
- `held` — проведено (есть отметки студентов, считается в прогрессе)
- `free` — СВОБОДНО (занятие отменено, не считается в прогрессе)

### 3.4 Оценки и их цвета
| Оценка | Значение | Цвет фона студента | Цвет кружка |
|--------|---------|-------------------|-------------|
| null | Не отмечен | серый (#f5f5f7) | серый |
| `0` | Присутствовал (без оценки) | зелёный (#e8f5e9) | зелёный |
| `5` | Отлично | красный (#ffebee) | красный |
| `4` | Хорошо | синий (#e3f2fd) | синий |
| `3` | Удовлетворительно | жёлтый (#fff8e1) | жёлтый |
| `2` | Неудовлетворительно | чёрный (#e0e0e0) | чёрный |
| `absent` | Отсутствовал | — | красный |

## 4. API (REST, английские роуты)

### 4.1 Группы и студенты
| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/groups` | Список групп |
| POST | `/api/groups` | Создать группу |
| GET | `/api/students?group_id=N` | Студенты группы |
| POST | `/api/students/bulk` | Массовое добавление |

### 4.2 Предметы
| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/subjects?group_id=N` | Список предметов (с прогрессом) |
| POST | `/api/subjects` | Создать предмет |
| GET | `/api/subjects/N/gradebook` | Ведомость по предмету |
| GET | `/api/subjects/N/substitution-list` | Предметы для замены (включая СВОБОДНО) |

### 4.3 Расписание
| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/schedule` | Всё расписание |
| POST | `/api/schedule` | Добавить пару в расписание |
| DELETE | `/api/schedule/N` | Удалить пару |
| GET | `/api/schedule/today` | Расписание на сегодня + проведённые занятия |

### 4.4 Занятия
| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/lessons` | Создать занятие (subject_id, actual_subject_id, status) |
| GET | `/api/lessons/N` | Детали занятия |
| PATCH | `/api/lessons/N/substitute` | Заменить предмет |
| PATCH | `/api/lessons/N/status` | Сменить статус |
| GET | `/api/lessons/N/attendance` | Отметки + студенты группы |
| POST | `/api/lessons/N/attendance` | Выставить оценку (student_id, grade) |

### 4.5 Отчёты
| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/reports/substitutions` | Все замены |
| GET | `/api/reports/average/N` | Средний балл по предмету |
| GET | `/api/reports/daily?date=YYYY-MM-DD` | Отчёт за день |
| GET | `/api/reports/neglected/N?min=3&days=14` | Студенты с пропусками |
| GET | `/api/students/N/grades?subject_id=N` | Оценки студента |

## 5. UI/UX (SPA на Vanilla JS)

### 5.1 Страницы
| Хэш | Страница | Назначение |
|-----|----------|-----------|
| `#home` | Главная | Сводка на сегодня (расписание + занятия), прогресс предметов, группы, управление |
| `#schedule` | Расписание | Редактирование шаблона расписания |
| `#subject/N` | Ведомость | Статистика, средний балл, таблица оценок |
| `#lesson/N` | Занятие | Выставление отметок студентам |
| `#students/N` | Студенты | Просмотр/добавление студентов группы |

### 5.2 Логика «Начать занятие»
1. Преподаватель нажимает на пару в расписании или кнопку «Начать занятие»
2. Всплывает диалог:
   - **✅ Проведено** → создаётся занятие со статусом `held`, открывается страница отметок
   - **🔄 Замена** → открывается список предметов группы + «СВОБОДНО» → при выборе создаётся занятие (с заменой или free)
3. Если занятие уже существует — по клику открывается его страница

### 5.3 Логика отметок
1. Тап на студенте циклически меняет оценку: `null → 0 → 5 → 4 → 3 → 2`
2. Цвет фона строки студента меняется в зависимости от оценки (см. 3.4)
3. Неотмеченные студенты — серый фон
4. Один тап = `0` (присутствовал) + зелёный фон

## 6. Типовые сценарии

### 6.1 День преподавателя
1. Открыть `#today` — посмотреть расписание
2. Нажать «Начать занятие» на первой паре
3. В диалоге выбрать «Проведено» или «Замена»
4. Проставить отметки студентам (тап-тап-тап)
5. Нажать «Готово» — вернуться на страницу сегодня

### 6.2 Замена предмета
1. Нажать на пару → диалог → «Замена»
2. Выбрать предмет из списка (или «СВОБОДНО» для отмены)
3. Занятие создаётся с фактическим предметом или статусом free

### 6.3 Контроль часов
1. На главной — прогресс-бары по каждому предмету (проведено/осталось)
2. На странице предмета — точные цифры: план, проведено, осталось

## 8. Правила разработки (из AGENT.md)
- Весь SQL — только в `database.py`
- Вертикальные срезы: один пункт → коммит → проверка
- Каждый срез тестируется локально перед push
- Запрещены: matplotlib, numpy, pillow, Kivy
- Сборка APK — только через GitHub Actions
