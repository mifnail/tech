## Пункт 5: Дубликаты занятий (несколько одинаковых в день)

### Проблема
- Можно создать несколько занятий одного subject_id на одну дату
- `startLesson` проверяет `existing = lessons.find(l => l.subject_id === e.subject_id && l.status === 'held')`
- Если уже есть held-занятие — показывает его, не даёт создать новое
- Счётчик занятий растёт, но отображается только одно
- Проблема: на странице расписания/сегодня поиск `lessons.find` находит только ПЕРВОЕ совпадение

### Стек изменений
- **database.py**: искать по `subject_id` + `date`, использовать `findLast` или выводить все совпадения
- **app.js**:
  - `_renderTodaySchedule` и `today()`: вместо `lessons.find(l => l.subject_id === e.subject_id && l.status === 'held')` использовать фильтр `lessons.filter(...)` и отображать ВСЕ найденные занятия
  - Добавить `lesson_number` в таблицу `lessons` для идентификации разных занятий
  - `schedule_today` эндпоинт может возвращать `lessons` группированными

### Вариант решения
1. Добавить поле `lesson_number` в таблицу `lessons` (через миграцию)
2. При создании занятия из расписания — передавать `lesson_number`
3. Отображать все занятия с одинаковым subject_id на одну дату

## Пункт 6: Экспорт отчётов не работает

### Проблема
- `window.open('/api/export/report/${today.date}.pdf')` открывает ссылку
- Если сервер не имеет `reportlab` или `openpyxl` — импорт guarded, но вызов экспортных функций упадёт
- В WebView `window.open` может не работать (блокируется)

### Стек изменений
- **app.js**: заменить `window.open` на программное скачивание через fetch + blob + download link
  ```js
  App.Download = {
    async as(name, url) {
      const res = await fetch(url);
      const blob = await res.blob();
      const a = document.createElement('a');
      a.href = URL.createObjectURL(blob);
      a.download = name;
      a.click();
      URL.revokeObjectURL(a.href);
    }
  };
  ```
- **report_export.py**: проверить что `export_report_pdf`/`export_report_xlsx` работают корректно
- Убедиться, что функции не падают при отсутствии библиотек (try/except)

- После генерации отчёта пользователь хочет сразу поделиться файлом через системный share (Telegram, WhatsApp, файловый менеджер)
- WebView не имеет доступа к нативному Android Share

### Стек изменений
- **Android (Java/Kotlin)**: Реализовать JavaScript Interface в WebView для вызова `Intent.ACTION_SEND` с файлом
- **Или**: Использовать Web Share API (navigator.share)
  - Chrome Android поддерживает `navigator.share({ files: [file] })` через File System Access API
  - Но в старых WebView может не работать
- **Вариант**: Скачивать файл и показывать кнопку "Поделиться" со ссылкой на файл
- **Вариант 2**: Сохранить файл в Download директорию через `<a download>` и уведомить пользователя "Файл сохранён"

### Приоритет
- Минимально: кнопка "Скачать" с `download` атрибутом
- Опционально: JS-интерфейс для нативного share
